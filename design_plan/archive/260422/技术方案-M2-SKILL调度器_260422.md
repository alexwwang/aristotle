## 一、模块概述

### 1.1 目标

将 SKILL.md 从「协议加载器」改造为「纯 MCP dispatcher」。所有四个命令（learn / reflect / sessions / review）统一通过 MCP `orchestrate_start` 路由，主 session 不再加载任何协议文件（REFLECT.md / REVIEW.md / LEARN.md / CHECKER.md）。

### 1.2 核心变更

| 维度 | Before (40行) | After (~60行) |
|------|--------------|---------------|
| 协议文件加载 | reflect→REFLECT.md, review→REVIEW.md | 零加载 |
| sessions 处理 | SKILL.md 内硬编码读 aristotle-state.json | MCP `orchestrate_start("sessions")` |
| reflect 参数解析 | REFLECT.md 内执行 | SKILL.md PRE-RESOLVE 区（~12行） |
| Review 交互 | REVIEW.md 内处理 | SKILL.md REVIEW FEEDBACK 区（~10行） |
| Action 类型 | fire_o, notify, done (3种) | fire_o, fire_sub, notify, done (4种) |

### 1.3 不变的部分

- Frontmatter（name, description, metadata）保持不变
- Parse Arguments 区格式不变，仅 reflect 路由目标变更
- 🦉 前缀通知规范不变
- MCP 工具名不暴露给用户的约束不变

### 3.4 PRE-RESOLVE 区（行 18-29，共 12 行）

**当前:** 无。

**目标** (12 行):
```markdown
## PRE-RESOLVE (reflect only)
Before calling MCP for reflect:
1. Call session_list() to get sessions.
2. Resolve target_session_id:
   - No argument → current session (first in list, or last active)
   - "last" → second entry in session_list
   - "session ses_xxx" → ses_xxx directly
   - "recent N" → entry at index N (1-based, excluding current)
3. If session_list fails or is empty: display "🦉 Could not resolve target session. Use /aristotle session ses_xxx" and STOP.
4. If "recent N" exceeds available: use closest available, display "🦉 Only M sessions available."
5. Detect user_language from recent messages (default: "en-US").
6. Get project_directory from current working directory.
7. Extract focus from --focus flag (default: "last").
```

**设计决策:**

| 决策点 | 选择 | 理由 |
|--------|------|------|
| target 解析位置 | SKILL.md（主 session） | MCP 无法调 session_list()（平台硬约束，§6.2） |
| user_language 检测 | 从最近消息推断 | 简单启发式，MCP 无上下文 |
| focus 默认值 | "last" | 与当前 REFLECT.md 行为一致 |
| 错误处理 | notify + STOP | 不调 MCP，直接展示用户友好消息 |

**边缘情况:**

| 情况 | 处理 |
|------|------|
| session_list 调用失败 | "🦉 Could not resolve target session." + STOP |
| session_list 返回空 | "🦉 No sessions found." + STOP |
| recent N 超出范围 | 取最近可用 + "🦉 Only M sessions available." |
| 无法确定 current session | session_list(limit=1) 取最新 |
| session ses_xxx 格式无效 | 透传给 MCP，由 MCP 返回错误 notify |

**行数:** 12 行（新增）

### 3.6 REVIEW FEEDBACK 区（行 44-53，共 10 行）

**当前:** 无。

**目标** (10 行):
```markdown
## REVIEW FEEDBACK
After displaying review content from MCP, show this menu and wait for user input:
```
Choose an action:
1. confirm — Accept all rules as-is
2. reject — Reject this reflection
3. revise N — Revise rule #N (append feedback after colon, e.g. "revise 1: add example")
4. re-reflect — Request deeper analysis
```
Map user input: "confirm"→call MCP `orchestrate_review_action(workflow_id, "confirm")` | "reject"→call MCP `orchestrate_review_action(workflow_id, "reject")` | "revise N: feedback"→call MCP `orchestrate_review_action(workflow_id, "revise", feedback, {"rule_index": N})` | "re-reflect"→call MCP `orchestrate_review_action(workflow_id, "re_reflect")`.
Execute the returned action per ACTION EXECUTION section.
```

**设计说明:**

1. **编号菜单格式** — 减少 LLM 解释错误。用户输入 "1"/"2"/"3"/"4" 或完整命令均可。
2. **workflow_id 来源** — 从 `orchestrate_start("review")` 返回值中获取，保存在上下文中。
3. **revise N 解析** — `N` 为 review 展示中的规则编号，`feedback` 为冒号后的用户意见。两者分别传入 MCP。
4. **re-reflect 循环防护** — MCP 层维护 `re_reflect_count`（max 3），SKILL.md 不关心。
5. **revise 返回 fire_o** — MCP 返回 `{action: "fire_o", o_prompt: "..."}`，O 完成后走 `o_done` handler。

**用户输入映射表:**

| 用户输入 | MCP 调用 | 预期返回 |
|----------|---------|---------|
| confirm | `orchestrate_review_action(wf_id, "confirm")` | `{action: "notify", message: "✅ N rules committed."}` |
| reject | `orchestrate_review_action(wf_id, "reject")` | `{action: "notify", message: "❌ Rule rejected."}` |
| revise N: feedback | `orchestrate_review_action(wf_id, "revise", feedback, {"rule_index": N})` | `{action: "fire_o", o_prompt: "..."}` |
| re-reflect | `orchestrate_review_action(wf_id, "re_reflect")` | `{action: "fire_sub", sub_prompt: "..."}` |

**行数:** 10 行（新增）

## 五、用户交互流程

### 5.1 `/aristotle` (reflect, 无参)

```
用户: /aristotle
  ↓

  ↓

  session_list() → [{ses_current, ...}]
  target_session_id = ses_current
  focus = "last"
  ↓

  ↓
MCP 返回: {action: "fire_sub", sub_prompt: "...", notify_message: "🦉 Aristotle launched..."}
  ↓
SKILL.md: 展示 "🦉 Aristotle launched..." → fire subagent
  ↓
[后台 R subagent 执行]
  ↓

  ↓

  ↓

  ↓
```
